User/customer studies

Putting customers at the center of the new solution

I create user-centric solutions, therefore I start by summarizing everything I learn about target customers inside a persona. Because I want the solution to appeal to users, I will base all my future design decisions on this persona.

Prototype creation

Tailoring the solutions to match business requirements

Creating a prototype is essential, as it allows me to visualize and test the website before it gets developed. I sketch and test the website navigation, information architecture, and many key features.

It is during this time that I test user flows, which are steps users take to accomplish specific tasks or goals within the website (subscribing to a newsletter, buying a product, or booking a consultation).

Accessibility

Making the solution accessible to users

I compare design colors, visual contrasts, typefaces, and font sizes against Web Accessibility Guidelines to ensure the content is perceivable, operable, understandable, and robust for all users, including those with disabilities.

User preferences

Anticipating users’ preferences

The final step of the design process consists in enhancing the experience for readability, eye comfort, and aesthetics in different lighting conditions or personal preferences. This allows the website to cater to users’ preferences for light or dark mode and further enhances customers’ browsing experience.

Website Development

Coding the website’s user interface

During the development process, I create a custom WordPress theme with the most up-to-date UI technologies, making the code light, fast, and responsive, ensuring a seamless experience across mobile, table, and desktop.
